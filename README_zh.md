# 📚 Academic Paper Analyzer - 学术论文 5 分钟速读分析器

> 基于 MinerU Cloud API 的高精度论文解析工具
> 将学术论文转化为精炼、流畅的技术文章
> **适合学术汇报、论文分享、快速掌握前沿研究**

## ✨ 核心特点

### 🎯 5 分钟速读体验
- **精炼内容**：将冗长论文提炼为 5 分钟可读的技术文章
- **自然叙述**：告别生硬的学术语言，用讲故事的方式解读论文
- **直觉优先**：从问题和直觉出发，而非堆砌技术细节

### 🔬 高精度解析
- **MinerU Cloud API**：业界领先的 PDF 解析精度
- **完整提取**：图片、表格、LaTeX 公式一个不漏
- **智能识别**：自动提取论文结构和元数据

### 📝 专业写作范式
- **分享优先**：适合学术汇报、公众号分享、技术博客
- **逻辑递进**：从简单到复杂，层层深入
- **生动比喻**：抽象概念具象化，让读者秒懂

## 🚀 快速开始

### 安装依赖

```bash
pip install requests markdown
```

### 获取 MinerU API Token

1. 访问 [MinerU 官网](https://mineru.net) 注册账号
2. 获取 API Token
3. 设置环境变量（推荐）：

```bash
export MINERU_TOKEN="your_token_here"
```

### 基本使用

```bash
# 第一步：解析 PDF
python scripts/mineru_api.py paper.pdf ./output

# 第二步：提取论文信息
python scripts/extract_paper_info.py ./output/*.md paper_info.json

# 第三步：由 Claude 智能生成文章（在 Claude Code 中执行）
# 阅读 paper_info.json 和 Markdown 文件，按照写作范式生成文章

# 第四步（可选）：生成 HTML 版本
python scripts/generate_html.py article.md output.html
```

## 🎨 写作范式

### ❌ 避免这样写

```markdown
## 方法

本文提出了一种基于哈希表的条件记忆模块。

主要创新点包括：
1. 可扩展的记忆库设计
2. 高效的查找机制
3. 动态记忆更新策略

在实验中，我们在多个数据集上验证了方法的有效性...
```

### ✅ 这样写更好

```markdown
## 为什么需要这个创新？

你有没有想过，大模型其实是没有记忆功能的？每次回答"中国的首都在哪里"，
它都要重新"计算"一遍答案，而不是像查字典一样直接找到。

这就像每次要知道一个单词的意思，都要从头推导词源，而不是直接翻字典。
这不是大炮打蚊子吗？

## 核心创新：记忆归记忆，计算归计算

Engram 的核心思想很简单：把那些可以"背下来"的知识，放到一个查表系统里...
```

### 关键原则

1. **从直觉切入**：先讲问题和动机，再讲技术
2. **讲历史背景**：介绍新技术前，先解释旧技术
3. **用简单例子**：选一个例子反复使用，贯穿全文
4. **生动比喻**：让抽象概念具象化（"大炮打蚊子"、"查字典 vs 背字典"）
5. **逻辑递进**：层层深入，让读者自然理解设计思路
6. **提炼洞见**：用一句话总结核心思想（"记忆归记忆,计算归计算"）
7. **充分配图**：每张关键图都要讲解到位

## 🎓 最佳实践

### 1. 节省上下文
- ✅ 信任 `paper_info.json` 中的元数据
- ✅ 直接引用图片路径，无需反复读取
- ✅ 只看 1-2 张核心架构图
- ❌ 不要反复读取所有图片文件

### 2. 写作风格
- ✅ 自然段落叙述，像讲故事
- ✅ 从直觉和问题出发
- ✅ 生动的比喻和简单的例子
- ❌ 避免分点列表（"1. 2. 3."）
- ❌ 避免 AI 常用词（"深入探讨"、"至关重要"）
- ❌ 避免机械化章节标题

### 3. 图片使用
- ✅ 公式用提取的图片，不用 LaTeX 语法
- ✅ 结果表用截图，比重新排版准确
- ✅ 每张关键图都要讲解
- ❌ 不要只贴 1-2 张框架图

## 📚 脚本说明

| 脚本 | 功能 | 用途 |
|------|------|------|
| `mineru_api.py` | MinerU Cloud API 调用 | 高精度 PDF 解析（推荐） |
| `convert_pdf.py` | 本地 PDF 转换 | 备选方案，需要 PyMuPDF |
| `extract_paper_info.py` | 提取论文元数据 | 生成结构化论文信息 |
| `generate_html.py` | Markdown → HTML | 生成 base64 嵌入图片的 HTML |

## ⚠️ 注意事项

### API 限制
- 单个文件最大 200MB
- 单个文件最多 600 页
- 支持 PDF、DOC、PPT、图片等格式

### 使用建议
- 优先使用 MinerU API，精度最高
- 不输出分析过程，只给用户最终文章
- 避免分点列表，使用自然段落
- 图片选择 3-5 张关键图表
